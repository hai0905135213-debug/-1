# 两校区距离功能｜白话说明

你不需要直接看代码。它现在做的是：

1. 把中央财经大学沙河校区和学院南路校区各记成一个固定坐标。
2. 高德为每家餐厅返回若干可能的分店，每个分店也有坐标。
3. 每个候选分店分别计算到两个校区的直线距离。
4. 导出时，一家店的每个候选分店都会带两项距离：

```json
"distanceKm": {
  "cufe_shahe": 1.955,
  "cufe_nanlu": 24.382
}
```

意思是：这个分店离沙河校区约 1.955 公里，离学院南路校区约 24.382 公里。

## 前端该怎么用

用户在页面上选择地区后，只需决定读取哪个字段：

| 用户选的地区 | 读取字段 |
| --- | --- |
| 沙河校区 | `distanceKm.cufe_shahe` |
| 学院南路校区 | `distanceKm.cufe_nanlu` |

如果页面不想显示全部分店，可以直接用 `nearestCandidateByCampus`：它已经分别找好了离两个校区最近的候选分店。

## 目前距离的含义

当前是“直线距离”，适合快速排序和筛选。它不是步行距离：中间可能有路口、围墙、河流或道路绕行。

以后若希望展示“步行约 X 分钟”或“驾车约 X 分钟”，再额外接高德路线规划 API；那是下一层功能，不影响现在的门店匹配和筛选。

## 以后更新数据时

每次新增餐厅并查完高德候选项后，依次运行：

```bash
node database/src/calculate-campus-distances.mjs
node database/src/export-for-app.mjs
```

校区地址一般不会变；只有要重新核验时，才运行：

```bash
node database/src/sync-campus-locations.mjs
```
