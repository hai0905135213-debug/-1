# 餐厅列表的下滑加载与图片懒加载

对应页面：`frontend/pages/restaurant-list/index.vue`。

## 它解决什么问题

餐厅数据库有 136 家店，且每家可能有外部地图图片。如果首页一次性请求全部图片，手机页面会慢、卡，也会浪费流量。

现在的规则是：

1. 每次只向后端要 12 家餐厅。
2. 用户滑到底部时，`onReachBottom` 才请求下一页。
3. 图片使用 uni-app 的 `lazy-load`，图片进入屏幕附近才下载。
4. 某张地图图片失败时，只把这一张换成占位图。
5. 用户切换沙河 / 学院南路校区后，从第一页重新加载；后端可以按对应校区距离排序。

## 后端正式接库后需要的返回格式

请求示例：

```text
GET /api/restaurants?page=1&pageSize=12&campus=cufe_shahe
```

返回示例：

```json
{
  "items": [
    {
      "id": "R_xxx",
      "name": "老张头东北菜",
      "cuisine": "东北菜",
      "fullAddress": "沙河高教园恒大城…",
      "photoUrl": "https://…",
      "recommendedDishes": "锅包肉，大骨头烩酸菜",
      "distanceKm": { "cufe_shahe": 1.955, "cufe_nanlu": 24.382 }
    }
  ],
  "hasMore": true
}
```

前端不需要自己计算距离，只按用户当前选择的校区读取对应的 `distanceKm` 字段。
