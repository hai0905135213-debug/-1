# 小群聊天记录数据清洗方案

## 目标

把小群聊天记录整理成可分析、可导入数据库的结构化样本，用来支持早期产品判断、标签体系、推荐规则和冷启动数据。

## 基本原则

- 原始聊天记录只放在 `data/raw/`，不直接进入业务数据库。
- 清洗后的数据放在 `data/processed/`。
- 进入数据库前必须脱敏。
- 早期只做内部产品分析，不展示原始聊天内容。

## 原始数据建议格式

可以先接受三类格式：

- 微信聊天导出的 `.txt`
- 人工整理的 `.csv`
- 人工整理的 `.xlsx`

建议原始字段：

| 字段 | 说明 |
| --- | --- |
| `message_time` | 发言时间 |
| `speaker` | 发言人原始标识 |
| `content` | 原始消息 |
| `source_group` | 来源群 |

## 清洗步骤

1. 导入原始聊天记录。
2. 删除系统消息、撤回提示、入群退群提示、纯表情和重复消息。
3. 删除或替换个人身份信息，包括姓名、手机号、微信号、学号、宿舍号。
4. 给发言人生成匿名哈希，例如 `user_001`、`user_002`。
5. 判断消息意图：
   - `meal_invite`：发起约饭
   - `meal_question`：询问有没有人吃饭
   - `meal_response`：响应别人
   - `meal_cancel`：取消或改期
   - `chat`：普通闲聊
6. 抽取结构化字段：
   - 饭点时间
   - 地点
   - 人数
   - 预算
   - 口味标签
   - 忌口或限制条件
   - 是否成局
   - 失败原因
7. 导出为 `csv` 或 `json`。
8. 人工抽查样本，确认脱敏和字段准确性。
9. 导入 `chat_message_samples` 表。

## 结构化样本字段

| 字段 | 说明 |
| --- | --- |
| `source_group` | 脱敏后的来源群 |
| `message_time` | 发言时间 |
| `speaker_hash` | 匿名发言人 |
| `clean_text` | 清洗后的消息 |
| `intent` | 消息意图 |
| `meal_time_text` | 饭点时间文本 |
| `location_text` | 地点文本 |
| `people_count` | 人数 |
| `budget_text` | 预算文本 |
| `taste_tags` | 口味标签 |
| `constraints` | 限制条件 |
| `matched_result` | 成局结果 |
| `failure_reason` | 失败原因 |

## 示例

原始消息：

```text
张三：今晚 6 点有人去二食堂吃麻辣烫吗，预算 25 左右，最好 3 个人。
```

清洗后：

```json
{
  "source_group": "group_001",
  "message_time": "2026-07-08 18:00:00",
  "speaker_hash": "user_001",
  "clean_text": "今晚 6 点有人去二食堂吃麻辣烫吗，预算 25 左右，最好 3 个人。",
  "intent": "meal_invite",
  "meal_time_text": "今晚 6 点",
  "location_text": "二食堂",
  "people_count": 3,
  "budget_text": "25 左右",
  "taste_tags": ["麻辣烫"],
  "constraints": [],
  "matched_result": "unknown",
  "failure_reason": ""
}
```

## 隐私边界

- 不保存原始姓名、微信号、手机号、学号、宿舍号。
- 不保存完整群名称，可以改成 `group_001`。
- 不公开展示原始聊天记录。
- 如果后续用于真实产品训练或运营分析，需要补充用户授权、数据删除和数据使用范围说明。
