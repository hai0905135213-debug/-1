# 接口草案

## 基础约定

- 基础路径：`/api`
- 响应格式：

```json
{
  "success": true,
  "data": {}
}
```

错误格式：

```json
{
  "success": false,
  "message": "错误原因"
}
```

## 健康检查

### GET `/api/health`

用于确认服务是否正常运行。

## 饭局接口

### GET `/api/meals`

获取可加入饭局列表。

返回核心字段：

- `id`
- `title`
- `mealTime`
- `location`
- `maxPeople`
- `currentPeople`
- `budget`
- `chatMode`
- `status`

### GET `/api/meals/:id`

获取饭局详情。

### POST `/api/meals`

创建饭局。

请求字段：

- `title`：饭局标题
- `mealTime`：饭点时间
- `location`：地点
- `maxPeople`：最大人数
- `budget`：预算
- `chatMode`：聊天模式
- `note`：补充说明

### POST `/api/meals/:id/join`

加入饭局。

### POST `/api/meals/:id/leave`

退出饭局。

## 资料接口

### GET `/api/profile/me`

获取当前用户资料。

### PUT `/api/profile/me`

更新当前用户资料。

请求字段：

- `nickname`
- `school`
- `tasteTags`
- `chatStyle`
- `bio`

## 评价接口

### POST `/api/reviews`

提交饭后评价。

请求字段：

- `mealId`
- `targetUserId`
- `score`
- `content`
