# UI 参考风格

## 参考来源

用户提供了一组可仿照的移动端美食社区界面截图，整体接近“美食推荐 + 探店 + 个人餐厅专辑”的产品气质。

## 整体风格

- 白底为主，界面干净，信息密度偏高。
- 主色使用深黑/藏青文字，强调色使用亮黄色。
- 页面标题很粗，视觉上有明显识别度。
- 食物图片占比较大，图片是核心吸引点。
- 卡片不做厚重阴影，更多依赖留白、圆角和分割线。
- 圆角整体偏克制，常见于图片、按钮、筛选胶囊和底部面板。
- 文案口语化，偏真实生活场景，不像传统工具软件。

## 首页方向

- 顶部包含定位、搜索框、地图或定位入口。
- 上方可放“精选专辑”横向滑动模块。
- 中部为推荐列表，支持地区、品类、排序、筛选四个筛选入口。
- 饭局/餐厅列表应采用左图右文布局：
  - 左侧大图。
  - 右侧标题、地点/距离/预算、推荐标签、用户短评。
  - 右侧可放收藏/想去按钮。

## 详情页方向

- 顶部使用大图或图片轮播，叠加推荐人头像和一句推荐语。
- 主体是白色圆角信息面板。
- 信息层级：
  - 名称大标题。
  - 推荐标签和榜单标签。
  - 品类、商圈、人均。
  - 营业/状态、设施、地址。
  - 推荐菜横向滑动。
  - 推荐官评价。
- 底部固定操作区可放：
  - 团购/拼饭入口。
  - 评价。
  - 想去/加入饭局主按钮。

## 找人页方向

- 顶部大标题“找人”，旁边放当前学校/城市下的推荐说明。
- 使用三个主 Tab：
  - 去过
  - 想去
  - 好评
- 下方使用横向品类筛选胶囊。
- 列表可做排行榜样式，展示头像、昵称、认证标识和数量。
- 对校园饭搭子平台来说，可改成：
  - 饭局活跃榜
  - 约饭响应榜
  - 好评饭搭子榜

## 动态页方向

- 顶部使用“关注 / 推荐”双 Tab。
- 下方是横向场景筛选：
  - 全部
  - 约会
  - 工作餐
  - 团建
  - 宴请
  - 度假
- 信息流卡片以用户头像、昵称、推荐标签、大图、短文、关联餐厅、点赞评论组成。
- 可迁移为校园饭搭子动态：
  - 饭后评价
  - 推荐食堂窗口
  - 饭局体验分享
  - 真实约饭反馈

## 我的页方向

- 顶部是个人信息区，带深青色背景和卡通形象。
- 展示关注、跟随/饭搭子、好评与想去等计数。
- 有“编辑资料”按钮。
- 中部“我的专辑”模块可改成“我的饭局”或“我的饭搭子专辑”。
- 空状态使用插画和明确操作按钮。

## 底部导航方向

参考截图是五栏：

- 首页
- 找人
- 中间加号发布
- 动态
- 我的

本项目建议调整为：

- 首页：饭局/推荐入口。
- 找人：饭搭子榜单和推荐同学。
- 发布：发起饭局。
- 动态：饭后评价和校园美食分享。
- 我的：资料、我的饭局、收藏、信用分。

## 对当前前端骨架的调整建议

- 将现有四栏 TabBar 改为五栏，中间保留发布入口。
- 新增 `pages/find/index.vue` 找人页。
- 新增 `pages/feed/index.vue` 动态页。
- 首页从“简单饭局列表”升级为“搜索 + 专辑 + 筛选 + 图文列表”。
- 饭局详情页引入大图头图和底部固定操作栏。
- 我的资料页升级为个人中心样式，保留资料编辑入口。

## 注意事项

- 可以借鉴布局和产品气质，但不要直接复制品牌元素、图标形象和具体图片。
- 项目应保留校园饭搭子的核心目标，不变成纯探店 App。
- 食物图片后续应使用真实校园食堂/周边餐厅图片，早期可先用占位图。
