# 功能和代码位置对照

这份文档用于晚上前端对接时快速找到接口对应代码。

## 后端入口

| 功能 | 文件 | 代码位置 |
| --- | --- | --- |
| 服务启动、路由分发、跨域处理 | `backend/src/server.js` | `http.createServer(...)`、`server.listen(...)` |
| 数据库连接、建表、CRUD | `backend/src/db.js` | `initTables()`、各导出函数 |
| 种子数据写入（首次启动） | `backend/src/db.js` | `seedIfEmpty()` |
| 统一返回 JSON | `backend/src/server.js` | `sendJson()` |
| 统一错误返回 | `backend/src/server.js` | `sendError()` |
| 登录态读取 | `backend/src/server.js` | `requireUser()` |

## 业务功能

| 页面/功能 | 接口 | 代码函数 |
| --- | --- | --- |
| 健康检查 | `GET /api/health` | 路由内直接返回 |
| 登录/注册 | `POST /api/auth/login` | `handleLogin()` |
| 我的资料页读取资料 | `GET /api/profile` | `handleGetProfile()` |
| 我的资料页保存资料 | `PUT /api/profile` | `handleUpdateProfile()` |
| 首页饭局列表 | `GET /api/meals` | `handleMealList()` |
| 发起饭局页提交 | `POST /api/meals` | `handleCreateMeal()` |
| 饭局详情页 | `GET /api/meals/:mealId` | `handleMealDetail()` |
| 加入饭局 | `POST /api/meals/:mealId/join` | `handleJoinMeal()` |
| 退出饭局 | `POST /api/meals/:mealId/leave` | `handleLeaveMeal()` |
| 我的饭局页 | `GET /api/meals/mine` | `handleMyMeals()` |
| 饭后评价 | `POST /api/meals/:mealId/reviews` | `handleCreateReview()` |
| 餐厅列表 | `GET /api/restaurants` | `handleRestaurantList()` |
| 餐厅详情 | `GET /api/restaurants/:id` | `handleRestaurantDetail()` |
| 新增餐厅 | `POST /api/restaurants` | `handleCreateRestaurant()` |

## 数据结构

当前使用 SQLite 持久化存储（`backend/data/fanfan.db`），数据库层代码在 `backend/src/db.js`。

| 数据 | 数据库表 | db.js 主要函数 |
| --- | --- | --- |
| 用户 | `users` | `getUser()` `createUser()` `getUserByStudentNo()` |
| 用户资料 | `profiles` | `getProfile()` `upsertProfile()` |
| 饭局 | `meals` | `getMeal()` `createMeal()` `listMeals()` `updateMeal()` |
| 饭局参与者 | `meal_participants` | `getParticipants()` `addParticipant()` `markParticipantLeft()` |
| 评价 | `reviews` | `getReviews()` `createReview()` |
| 餐厅 | `restaurants` | `getRestaurant()` `listRestaurants()` `createRestaurant()` |

后续切 MySQL 时，接口字段不变，只需替换 db.js 中的 SQLite 驱动为 MySQL 驱动。
