# 功能和代码位置对照

这份文档用于晚上前端对接时快速找到接口对应代码。

## 后端入口

| 功能 | 文件 | 代码位置 |
| --- | --- | --- |
| 服务启动、路由分发、跨域处理 | `backend/src/server.js` | `http.createServer(...)`、`server.listen(...)` |
| 内存数据初始化 | `backend/src/server.js` | `seedData()` |
| 统一返回 JSON | `backend/src/server.js` | `sendJson()` |
| 统一错误返回 | `backend/src/server.js` | `sendError()` |
| 登录态读取 | `backend/src/server.js` | `requireUser()` |

## 业务功能

| 页面/功能 | 接口 | 代码函数 |
| --- | --- | --- |
| 健康检查 | `GET /api/health` | 路由内直接返回 |
| 登录/注册 | `POST /api/auth/login` | `handleLogin()` |
| 我的资料页读取资料 | `GET /api/profile` | `handleGetProfile()` |
| 我的资料页保存资料 | `PUT /api/profile` | `handleUpdateProfile()` |
| 首页饭局列表 | `GET /api/meals` | `handleMealList()` |
| 发起饭局页提交 | `POST /api/meals` | `handleCreateMeal()` |
| 饭局详情页 | `GET /api/meals/:mealId` | `handleMealDetail()` |
| 加入饭局 | `POST /api/meals/:mealId/join` | `handleJoinMeal()` |
| 退出饭局 | `POST /api/meals/:mealId/leave` | `handleLeaveMeal()` |
| 我的饭局页 | `GET /api/meals/mine` | `handleMyMeals()` |
| 饭后评价 | `POST /api/meals/:mealId/reviews` | `handleCreateReview()` |

## 数据结构

当前 MVP 为了方便联调，先使用内存数据：

| 数据 | 变量 | 未来数据库表 |
| --- | --- | --- |
| 用户 | `users` | `users` |
| 用户资料 | `profiles` | `profiles` |
| 饭局 | `meals` | `meal_posts` |
| 饭局参与者 | `participants` | `meal_participants` |
| 评价 | `reviews` | `reviews` |

后续接 MySQL 时，建议保留接口字段不变，只替换这些 Map 的读写逻辑。

## 已在代码里标注的功能

`backend/src/server.js` 中每个主要接口函数前都有 `// 功能：...` 注释，方便你在编辑器里搜索。

可搜索关键词：

- `功能：用户登录`
- `功能：饭局列表`
- `功能：发布饭局`
- `功能：加入饭局`
- `功能：我的饭局`
- `功能：饭后评价`
