## 2026-07-07

### 已读取的项目资料

- 读取了 `校园饭搭子平台_完整申报书 人工润色版.docx`。
- 项目定位：面向高校学生的校园饭搭子智能匹配平台。
- 核心价值：不是单纯交友软件，而是“为了吃好这顿饭，顺手找个合适的人一起吃”的拼餐撮合工具。

### 第一阶段产品方向

第一阶段先做 MVP，不做大而全的平台。

MVP 核心闭环：

1. 用户登录并完善基础资料
2. 用户发起一个饭局
3. 其他用户浏览并加入饭局
4. 人数满足后成局
5. 线下就餐后进行简单评价

第一阶段暂缓：

- 复杂课表联动
- 商家抽佣
- PRO 会员
- 大规模推荐算法
- 多学校扩展
- 完整聊天系统

### 初步页面规划

- 首页：展示当前可加入饭局、推荐饭局、快捷发起入口
- 发起饭局页：填写吃什么、时间、地点、人数、预算、聊天模式
- 饭局详情页：查看发起人信息、参与人数、加入/退出饭局
- 我的饭局页：查看我发起的、我加入的、待确认的饭局
- 我的资料页：学生认证、口味偏好、信用分、历史评价

### 初步技术架构

- 前端：微信小程序或 uni-app
- 后端：Node.js + Express
- 数据库：MySQL
- 部署：腾讯云/阿里云轻量服务器

### 初步数据表

- `users`：用户表
- `profiles`：用户资料表
- `meal_posts`：饭局表
- `meal_participants`：饭局参与者表
- `reviews`：评价表
- `reports`：举报/爽约记录表
- `restaurants`：店铺/食堂窗口表

## 2026-07-09

### 后端 MVP 骨架

- 用户说明：前端代码已有一部分但暂未上传，当前优先处理后端。
- 已创建 `backend/` 后端目录。
- 技术取舍：当前先使用 Node.js 原生 HTTP 服务，不引入外部依赖，避免安装依赖和网络环境影响联调。
- 当前数据暂存在内存 Map 中，字段按后续 MySQL 表结构预留。
- 已支持跨域请求，方便前端本地联调。
- 后端默认监听 `127.0.0.1:3001`，避免本机联调时暴露到局域网。

### 已完成接口

- `GET /api/health`：健康检查
- `POST /api/auth/login`：登录/注册，返回 MVP token
- `GET /api/profile`：获取当前用户资料
- `PUT /api/profile`：更新用户资料
- `GET /api/meals`：饭局列表，支持状态、校区、关键词、可加入筛选
- `POST /api/meals`：发布饭局
- `GET /api/meals/:mealId`：饭局详情
- `POST /api/meals/:mealId/join`：加入饭局
- `POST /api/meals/:mealId/leave`：退出饭局
- `GET /api/meals/mine`：我的饭局
- `POST /api/meals/:mealId/reviews`：发布评价
